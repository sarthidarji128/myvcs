myvcs

A Minimal Local Version Control System

What it Solves

Traditional VCS tools like Git often need network access for cloning, pushing, pulling, or remote tracking. myvcs works completely offline—ideal for air‑gapped machines, unreliable connections, or local‑only projects.

Key Features

  • Project setup & management (register, switch, list, remove)
  • Initialize a .myvcs/ repository in any directory
  • Commit snapshots with messages
  • View commit log
  • Status, diff, and revert commands
  • Efficient object storage using SHA‑1 hashes

Quick Start

# Clone
git clone https://github.com/sarthidarji128/myvcs.git
cd myvcs

# Make script executable (if needed)
chmod +x myvcs.sh

# Initialise your first project
./myvcs.sh setup myproject /path/to/your/project

# Basic workflow
./myvcs.sh use myproject
# edit files …
./myvcs.sh status
./myvcs.sh diff my_file.txt
./myvcs.sh commit "Describe changes"
./myvcs.sh log
./myvcs.sh revert c1   # revert to a previous commit

Project Structure

  • myvcs.py – core Python engine
  • myvcs.sh – Bash CLI wrapper
  • .myvcs_config.json – generated project registry
  • .myvcs/ – per‑project repo (objects/, commits.json)
